# План: приложение «намаз в чёлке» для macOS (рабочее название — **Miqat**)

> **Miqat** (ميقات) — «установленное время поклонения». Название рабочее, поменяем когда захочешь.
>
> Суть продукта одной строкой: **всегда видимый в чёлке MacBook обратный отсчёт до следующего намаза** — красиво, приватно (считается на устройстве, без рекламы и аккаунтов), оффлайн.

---

## 📌 Как мы работаем вместе (прочитай первым)

Ты **не кодишь**. Продукт делаем в паре:

| Что делаю **я (Claude)** | Что делаешь **ты** |
|---|---|
| Пишу и правлю весь Swift-код прямо в файлах | Открываешь проект в Xcode, жмёшь ⌘R (Запустить) |
| Объясняю каждый шаг простым языком | Смотришь результат на своём экране / чёлке |
| Чиню ошибки | Присылаешь мне скриншот или **текст красной ошибки** из Xcode |
| Предлагаю решения и варианты | Принимаешь решения: как выглядит, что важно, что дальше |
| Держу этот план в актуальном виде | Тестируешь и проверяешь, что времена намаза **совпадают с твоей мечетью** |

**Важно честно:** я не вижу твой экран и не могу нажать «Запустить» за тебя. Поэтому цикл всегда такой:
> ты описываешь / решаешь → я пишу код → ты жмёшь ⌘R → говоришь что видишь (или кидаешь ошибку) → я правлю. Повторяем.

Лучший режим: держать эту папку открытой в Claude Code (уже так и есть) + Xcode рядом.

**Делаем (v1):**
- Один экран ценности: следующий намаз + отсчёт в чёлке.

- Коран, тасбих, поиск мечетей, лунный календарь целиком.
- Аккаунты, облако, соцфичи.

> Правило: **одна платформа (Mac) · одна аудитория (мусульмане) · одна фича (отсчёт намаза)**. Расширяемся только после первых живых пользователей.

## Этапы сборки — майлстоуны

Каждый майлстоун маленький и **проверяемый на экране**. Ниже — что строим, что делаешь ты, и «готово когда».

### M0 — Пустое приложение запускается *(1 сессия)*
- **Строим:** пустой macOS-проект на SwiftUI, открывается окошко «Hello».
- **Ты:** ставишь Xcode → я создаю проект → ты жмёшь ⌘R.
- **Готово когда:** на экране появилось окно приложения. *(Это доказывает, что вся кухня работает.)*

### M1 — Времена намаза в обычном окне *(1–2 сессии)*
- **Строим:** подключаем библиотеку **Adhan** (проверенный open-source расчёт времён намаза). Показываем в обычном окне: 5 времён на сегодня, какой намаз следующий, отсчёт до него. Координаты пока «зашиты» (например, твой город).
- **Ты:** запускаешь, сверяешь времена глазами.
- **Готово когда:** видишь корректные 5 времён и тикающий отсчёт. *(Сердце продукта уже работает — просто ещё не в чёлке.)*

### M2 — Геолокация + метод расчёта *(1–2 сессии)*
- **Строим:** берём реальные координаты через CoreLocation (с обработкой разрешения), считаем времена для твоего фактического места. Добавляем выбор **метода расчёта** и **мазхаба для Асра** (см. раздел «Точность» — это про доверие).
- **Ты:** разрешаешь геолокацию, проверяешь, что времена = времена твоей мечети/IslamicFinder.
- **Готово когда:** времена совпадают с местным источником, при смене метода меняются правильно.

### M3 — UI в чёлке: свёрнутое состояние *(2–3 сессии)*
- **Строим:** окно без рамки, поверх всех окон, приклеенное к чёлке сверху по центру. В нём тонкая «пилюля»: иконка + «Магриб · 4:32».
- **Ты:** запускаешь, смотришь у своей чёлки, говоришь по пикселям что подвинуть.
- **Готово когда:** у чёлки всегда висит аккуратный отсчёт, не мешает.

### M4 — Разворот/сворачивание + полная панель *(2–3 сессии)*
- **Строим:** по наведению/клику чёлка плавно «вырастает» в панель: 5 времён списком (следующий подсвечен), крупный отсчёт, дата по Хиджре, стрелка Киблы. Клик мимо — сворачивается.
- **Ты:** тестишь анимацию и удобство.
- **Готово когда:** разворот плавный и приятный, вся ключевая инфа на месте.

### M5 — Напоминания *(1–2 сессии)*
- **Строим:** локальное уведомление за N минут до намаза и в момент наступления. Визуальная подсветка у чёлки. Звук азана — **опция** (по умолчанию тихо: на рабочем маке громкий азан неуместен).
- **Ты:** проверяешь, что напоминает вовремя и деликатно.
- **Готово когда:** приходит корректное напоминание, звук настраивается.

### M6 — Полировка *(2–4 сессии)*

## 🕌 Точность времён намаза = доверие (не пропускать!)

Мусульмане не будут пользоваться приложением, если времена «немного не те». Поэтому с самого начала закладываем правильность:

- **Метод расчёта** (угол зари/сумерек различается): Muslim World League, ISNA, Umm al-Qura (Саудия), Egyptian, Karachi, Tehran и др. Дадим выбор + разумный дефолт по региону.
- **Мазхаб для Асра:** ханафитский (тень 2×) vs стандартный/шафи (тень 1×). Обязательно выбор.
- **Высокие широты** (короткие/длинные ночи): правило для регионов, где солнце «не заходит» как надо.
- **Часовой пояс и переход на летнее время** — берём из системы, но проверяем.
- **Валидация:** на этапе M2 сверяем наши времена с местной мечетью или IslamicFinder. Это наша страховка доверия.

Всё это уже умеет библиотека **Adhan** — нам надо только правильно её настроить и дать выбор в UI.

## Этап M2 — Автолокация (план)

Цель: убрать жёстко зашитый Грозный — приложение само определяет город и считает времена для него.

### Готовые бесплатные решения (разведка)
- **CoreLocation** (Apple, встроено) — координаты (Mac по WiFi). Бесплатно, без ключа, приватно (не уходит наружу), оффлайн после первого фикса. Требует **бандла .app** + разрешение пользователя.
- **CLGeocoder** (Apple, встроено) — координаты → город + часовой пояс.
- **Adhan** (уже стоит) — расчёт времён из координат + метод.
- Альтернативы: IP-геолокация (мгновенно, без бандла, но онлайн и ~город точность); Aladhan API (времена+метод по координатам, но онлайн — против «оффлайн/приватно»). Не берём в основу.

### Два подвоха
1. CoreLocation требует «настоящего» **.app с Info.plist** (ключ `NSLocationWhenInUseUsageDescription`). Голый SPM-бинарь разрешение не получит.
2. Метод расчёта под регион выбираем сами — Adhan по стране метод не подбирает. Сейчас точная калибровка под муфтият Грозного; для других городов нужен дефолт + ручной выбор.

### Шаги
- **Шаг 0 — упаковка в `.app`** (prereq; нужен и для релиза): `Info.plist` (разрешение геолокации, bundle id, `LSUIElement`), сборочный скрипт (`packaging/build_app.sh`) заворачивает бинарь в `Miqat.app` + ad-hoc подпись.
- **Шаг 1 — CoreLocation:** запросить разрешение, получить координаты; при отказе — откат (ручной город / последняя известная / Грозный); кэшировать локацию.
- **Шаг 2 — CLGeocoder:** координаты → название города в шапку + правильный пояс.
- **Шаг 3 — метод:** дефолт по региону (рядом с Грозным — наша калибровка; иначе стандартный, напр. MWL).
- **Шаг 4 (позже) — настройки:** ручной выбор города/метода/мазхаба + сохранение.

Решения: способ локации — **CoreLocation** (правильно, бандл всё равно нужен). Метод для чужих городов — дефолт MWL на старте, настройки позже.
