# Карта проекта Miqat

Кратко: из чего собрано приложение, зачем каждый кусок, и как всё работает.

## 1. Стек — и зачем каждый

| Технология | Зачем нужна |
|---|---|
| **Swift** | Язык. Единственный нативный путь для macOS и работы с чёлкой. |
| **SwiftUI** | Рисуем интерфейс (пилюля, панель, список, кольцо) — декларативно и современно. |
| **AppKit** (`NSPanel`, `NSScreen`, `NSEvent`) | Низкоуровневое управление окном: посадить у выреза, держать поверх всего, ловить мышь. SwiftUI сам так не умеет; мостик между ними — `NSHostingView`. |
| **Swift Package Manager** | Сборка + зависимости. Подключает Adhan одной строкой; проект открывается в Xcode. |
| **Adhan** (сторонняя либа) | Считает времена намаза и направление Киблы **на устройстве**, оффлайн. Самую сложную часть — астрономию — не пишем сами. |
| **Combine / `@Published`** | Реактивность: поменялась модель → интерфейс сам перерисовался. Не дёргаем UI вручную. |
| **`Timer`** | Тик раз в секунду — живой обратный отсчёт. |
| **`NSEvent` global monitor** | Определяем наведение по позиции мыши (а не через `onHover` — так нет мерцания). |
| **Приватный CGS API** (`CGSSpace`) | Держит окно на **всех** рабочих столах и поверх полноэкранных приложений. Публичного аналога нет. |

## 2. Архитектура — слоями

Принцип: **каждый слой знает только про соседний**. Логика расчёта не знает про интерфейс; интерфейс ничего не вычисляет — только показывает.

```
MiqatApp            точка входа: запускает окно + иконка в меню (выход)
   │
   ▼
NotchController     ОКНО: размер/позиция у чёлки, все Spaces, логика наведения
 + CGSSpace         (open/close), общее состояние NotchState
   │
   ▼
NotchView (SwiftUI) ВИД: контейнер-морф пилюля↔панель, список, кольцо, компас, узор
   ▲
   │  читает данные
ClockModel          СЕРДЦЕБИЕНИЕ: тикает 1 раз/сек, хранит now / времена / следующий / прогресс
   ▲
   │
PrayerEngine        МОЗГ: времена намаза + Кибла через Adhan, параметры Грозного
```

Плюс два помощника сбоку: **Design** (цвета + формат времени/даты) и **CGSSpace** (обёртка приватного API).

## 3. Файлы — кто за что отвечает

| Файл | Роль |
|---|---|
| `Package.swift` | Манифест сборки: имя, зависимость Adhan. |
| `MiqatApp.swift` | Запуск приложения; создаёт `NotchController`; иконка 🌙 в меню с «Выйти». Прячет иконку из дока (`.accessory`). |
| `NotchController.swift` | Окно `NSPanel` у чёлки: позиция, «все Spaces», монитор мыши, открыть/закрыть. Здесь же `NotchState` — общий флаг `expanded` и размеры. |
| `NotchView.swift` | Весь визуал (SwiftUI): `NotchContainer` (морфинг), `CollapsedPill`, `ExpandedPanel`, `PrayerRow`, `ProgressRing`, `QiblaCompass`, `StarPattern`. |
| `PrayerEngine.swift` | Расчёт: 5 времён, следующий намаз, предыдущий (для кольца), Кибла. Параметры Грозного (углы + поправки муфтията). |
| `ClockModel.swift` | Таймер 1 сек; публикует `now`, `slots`, `next`, `progress`. |
| `Design.swift` | Палитра (золото/кремовый/фон) + форматирование времени, отсчёта, даты по Хиджре. |
| `CGSSpace.swift` | Обёртка приватного Spaces API (MPL-2.0). |

## 4. Как это работает — два потока

**Поток времени (отсчёт живой):**
```
Timer (1 сек) → ClockModel пересчитывает now/next/progress
             → @Published меняется → SwiftUI сам перерисовывает цифры
```

**Поток наведения (пилюля ↔ панель):**
```
Мышь двигается → global monitor → NotchController.evaluateHover()
   курсор в зоне выреза?  → NotchState.expanded = true
   курсор ушёл?           → через 0.18 с → expanded = false
→ SwiftUI морфит контейнер пружинной анимацией
```

Окно при этом **не меняет размер** (фиксированное, клики проходят насквозь) — «растёт» только контент внутри. Поэтому анимация плавная.

## 5. Почему так, а не иначе (ключевые решения)
- **SPM, а не полноценный Xcode-проект** — я могу править всё сам, ты просто жмёшь ▶︎.
- **Окно фиксированного размера + морфинг в SwiftUI** — вместо резайза окна: плавно и не блокирует клики.
- **Наведение по позиции мыши** — вместо `onHover`: убирает мерцание.
- **Приватный CGS API** — единственный способ надёжно держать виджет на всех столах (цена — только прямая раздача, не App Store).
- **Adhan on-device** — приватно, оффлайн, без сервера.
